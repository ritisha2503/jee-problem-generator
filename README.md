# MathGen-RAG Dashboard

An interactive, local Retrieval-Augmented Generation (RAG) workspace designed to synthesize **JEE Advanced conceptual mathematics problems** and verification paths. The system leverages local vector databases, lightweight Hugging Face embeddings, and multi-model router APIs (Gemini, Kimi, Qwen) to generate highly rigorous, context-aware math questions with beautiful LaTeX formatting.

---

## 🚀 Features

* **Local Vector Ingestion:** Pipeline built with `langchain-huggingface` and `ChromaDB` running local `BAAI/bge-small-en-v1.5` embeddings (384-dimension schema).
* **LaTeX-Aware Semantic Chunking:** Uses a customized `RecursiveCharacterTextSplitter` configured with LaTeX specific regex blocks (`$$`, `\\[`, `\\begin{equation}`) to keep equations intact during splitting.
* **Dynamic Multi-Model Workspace:** Dropdown switching architecture allowing instant synthesis routing to **Gemini 2.5 Flash**, **Kimi Chat**, or **Qwen Long-Context**.
* **Beautiful UI/UX Render Platform:** Fast, responsive frontend styled with Tailwind CSS, supporting automated markdown-to-HTML compilation and robust asynchronous **MathJax v3** typesetting for seamless inline `$ ... $` and display math layout elements.

## 🛠️ Installation & Setup
Ensure you have Python 3.11+ and the fast modern dependency manager uv installed.

1. Clone & Navigate to Project Room
```bash
git clone <your-repository-url>
cd rag-jee
```
2. Configure Environment Secrets
Create a .env file in the root directory and append your provider credentials:

```python
GEMINI_API_KEY="your_gemini_api_key_here"
KIMI_API_KEY="your_kimi_api_key_here"
QWEN_API_KEY="your_qwen_api_key_here"
```

3. Synchronize Dependencies
Install the required system structures natively inside a localized virtual environment:

```bash
uv add fastapi uvicorn langchain langchain-community langchain-huggingface langchain-chroma pypdf python-dotenv torch
```

## 📈 Database Initialization (Ingestion Pipeline)

Before launching the web system dashboard, drop your reference math PDFs or chapter text logs into the ./data/ directory, then initialize your vector store setup:

```bash
# Clean out any conflicting dimension history and ingest source documents
rm -rf chroma_db
uv run python -c "from app.rag import index_pdfs; print(index_pdfs())"
```

## 💻 Running the Dashboard

Launch the hot-reloading production simulation environment on your local server stack:

```bash
uv run uvicorn main:app --reload
```

Once initialized, open your browser and access the workspace dashboard at:
👉 http://127.0.0.1:8000
